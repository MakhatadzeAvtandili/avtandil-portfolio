Portfolio Documentation

Comprehensive guide for setting up, customizing, and deploying the Avtandili Makhatadze Professional Portfolio website.

This project represents a cutting-edge, bilingual (English/Georgian) portfolio website built with a modern tech stack including React, Vite, Tailwind CSS, and Framer Motion. It aims to provide a visually stunning, highly performant, and professional platform ("Summer Breeze" theme) for showcasing skills and projects effectively.

Tech Stack

Core Framework: React 18+
Build Tool: Vite
Styling: Tailwind CSS v4, @tailwindcss/typography
Routing: React Router DOM v6
Animations: Framer Motion
i18n: react-i18next, i18next
SEO: react-helmet-async
Contact Form: EmailJS

Project Structure

The project adheres to a clean and organized Vite + React structure:

Features & Implementation Details

Routing (React Router v6)

Utilizes BrowserRouter with routes defined in App.jsx. Includes dynamic routes like /blog/:slug. The Header component intelligently switches between <a> tags for smooth scrolling on the homepage and <Link> tags for cross-page navigation, ensuring optimal user experience.

Internationalization (react-i18next)

Fully supports English (en) and Georgian (ka). Translations are managed via JSON files in src/i18n/locales/. User language preference is persisted using localStorage. The useTranslation hook provides easy access to the t function and i18n instance.

Animations (Framer Motion)

Sophisticated animations are implemented using Framer Motion:

  • Preloader: A unique SVG logo transformation animation provides a high-class loading experience (Preloader.jsx). Uses layoutId for seamless transition potential.
  • Staggered Entrance: Components like workflow steps, project cards, and skills fade in sequentially using whileInView and transition delays.
  • Micro-interactions: Subtle hover effects (scaling, translation) on cards, buttons, and skill icons enhance user feedback.
  • Layout Animations: Used for the active filter pill in the Projects section.

Styling (Tailwind CSS v4 & Typography)

Tailwind CSS v4 is used via the Vite plugin with configuration inside src/index.css (@theme). The design adheres to the "Summer Breeze" light theme. Animated background blobs add subtle visual interest. The @tailwindcss/typography plugin provides excellent default styling for blog post content via the prose classes.

EmailJS Contact Form

The Contact.jsx form integrates @emailjs/browser for direct email sending. Remember to replace placeholder keys (YOUR_SERVICE_ID, YOUR_TEMPLATE_ID, YOUR_PUBLIC_KEY) with your actual EmailJS credentials in the code. Basic validation is included.

SEO (react-helmet-async)

The SEO.jsx component dynamically manages page titles, meta descriptions, and social sharing tags (Open Graph, Twitter Cards) for optimal search engine visibility and link previews using react-helmet-async.

Local Setup Instructions

Follow these steps carefully to run the project locally.

Prerequisites

  • Node.js (v18 or later)
  • npm (v8 or later)
  • Git

1. Clone Repository


git clone  portfolio-react
cd portfolio-react

2. Install Dependencies


npm install

3. Configure EmailJS (Critical Step!)

Open src/components/Contact.jsx and update the following variables inside the sendEmail function with your EmailJS account details:


const serviceID = 'YOUR_SERVICE_ID';   // Replace with your Service ID
const templateID = 'YOUR_TEMPLATE_ID';  // Replace with your Template ID
const publicKey = 'YOUR_PUBLIC_KEY';   // Replace with your Public Key (User ID)

Info:

Make sure your EmailJS template uses the variables from_name, from_email, and message.

4. Run Development Server


npm run dev

Access the site at the URL provided in the terminal (usually http://localhost:5173).

Customization Guide

Tailor the portfolio to your personal brand:

  • Content (Texts): Modify the JSON files in src/i18n/locales/ for both English and Georgian.
  • Images: Replace src/assets/portrait.jpg. Update image paths in src/data/projects.js, src/data/posts.js, and src/data/testimonials.js.
  • Data Files: Update the content within src/data/*.js files (projects, posts, testimonials) with your information. Ensure unique slug values.
  • Skills: Edit the skills array in src/components/Skills.jsx.
  • Social Links: Update URLs in src/components/Hero.jsx and src/components/Footer.jsx.
  • Theme Colors: Adjust CSS variables (--color-primary, etc.) in src/index.css under @theme.

Build for Production

Generate optimized static files for deployment:


npm run build

The output will be created in the dist/ folder.

Deployment Guide: Going Live

Deploy the contents of the dist/ folder to any static host.

Option 1: Netlify (Recommended)

  1. Push your code to a Git provider (GitHub, GitLab, etc.).
  2. Connect your Git provider to Netlify.
  3. Select your repository.
  4. Configure build settings:
    • Build command: npm run build
    • Publish directory: dist
  5. Deploy. Netlify automatically handles SPA routing for Vite. If needed, add a public/_redirects file with /* /index.html 200.

Option 2: Vercel

  1. Push code to Git provider.
  2. Connect Git provider to Vercel.
  3. Import your repository. Vercel auto-detects Vite settings (npm run build, output dist).
  4. Deploy. Vercel automatically handles SPA routing.

Option 3: Other Static Hosts

Upload the dist/ folder contents. Crucially, you must configure the server to handle SPA routing by redirecting all requests to index.html. Consult your hosting provider's documentation for specific instructions (e.g., using .htaccess for Apache or Nginx configuration).


# Example Nginx configuration snippet for SPA
server {
    # ... other server configs ...
    location / {
        try_files $uri $uri/ /index.html;
    }
}
